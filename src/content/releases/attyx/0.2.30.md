# Attyx v0.2.30

## Smarter Command Execution in Tabs and Splits

When you open a new tab or split with `--cmd`, the command now runs inside a fully initialized shell session. Previously, commands launched via `$SHELL -c` which skipped your shell config — meaning tools installed through your `.zshrc`, `.bashrc`, or other dotfiles weren't available.

Now Attyx passes the command through shell integration so your full PATH, aliases, and environment are loaded before the command runs. This works across zsh, bash, fish, and nushell.

```
attyx split vertical --cmd 'claude -p "tell me a joke"'
attyx run 'npm start'
```

Both examples now behave exactly as if you'd typed the command into a fresh interactive shell.

## Improved IPC Help and Scripting

- All IPC subcommand help texts have been rewritten with clearer descriptions, argument details, and practical examples — designed to be useful for both humans and AI agents reading `--help` output.
- Help output now prints to stderr, so it doesn't interfere with stdout when scripting or piping IPC results.

## Reliability

- IPC control sockets are now explicitly cleaned up on macOS app exit, preventing stale socket files from accumulating.
