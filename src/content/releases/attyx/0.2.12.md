# Attyx v0.2.12

## Seamless Daemon Upgrades

Updating Attyx no longer requires manually restarting the daemon. When a new client detects a version mismatch, the daemon automatically upgrades itself in place — your sessions, tabs, splits, and running processes all survive the transition. No data loss, no interruption.

If you're upgrading from a version that predates this feature, you'll see a one-time notification prompting you to run `attyx kill-daemon`. After that, all future upgrades are automatic.

## Cleaner Session Replay

Reconnecting to an existing session (after switching tabs, reattaching, or upgrading) no longer occasionally shows garbled text or stray numbers on screen. Previously, the replay buffer could split an escape sequence at its boundary, causing fragments like color codes to render as visible characters.

## Smoother Tab Switching

Fixed a visual glitch that could appear when switching between tabs in session mode.

## Bug Fixes

- Fixed a crash when running `attyx kill-daemon` while a client was connected.
- Improved daemon reconnect reliability — stale socket data is now properly drained before reattaching.
