# Attyx v0.2.20

## Fixes

- **Linux CPU usage** — fixed high idle CPU consumption on Linux caused by the render loop treating every blink timer tick as a dirty frame. The renderer now only redraws when the blink state actually changes.

- **Updater version display** — the update dialog now correctly shows the current running version instead of a stale value, so you can tell at a glance what you're upgrading from.

- **Daemon stability** — fixed several bugs that could cause the daemon to hang or lose track of sessions:
  - Session counter now stays in sync with actual slot occupancy, preventing "max sessions reached" errors when slots were available.
  - Killing a session no longer prematurely frees its slot — dead sessions are preserved as recent entries until explicitly removed.
  - Reviving a dead session no longer double-counts it in the session tracker.
  - Multi-pane sessions now correctly detect when all panes have exited, instead of only checking the first pane.
  - Clients that send oversized or malformed messages are now disconnected cleanly instead of causing the daemon to spin.
