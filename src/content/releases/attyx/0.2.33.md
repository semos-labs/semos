# Attyx v0.2.33

## Safer Daemon Upgrades

Upgrading Attyx no longer risks losing your sessions. Previously, if the new binary failed to start during a daemon upgrade, the old daemon would already be gone — leaving you with no daemon, no sessions, and no way to reconnect.

Now the old daemon stays alive as a safety net. It launches the new version, verifies that all sessions transferred successfully, and only then exits. If anything goes wrong — the new binary crashes, sessions go missing — the old daemon rolls back and keeps serving as if nothing happened.

If the upgrade is interrupted (e.g. a crash mid-handoff), the next launch detects orphaned session metadata and restores your tabs, splits, and working directories as reconnectable sessions.
