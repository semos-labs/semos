# Attyx v0.2.39

## Pane IDs That Don't Move

**Stable pane IDs** — Every pane now gets a permanent numeric ID that never changes, even when other panes are closed. Previously, pane targeting used `tab.pane` indices that shifted whenever the pane tree restructured — meaning a script targeting `-p 1.2` could hit the wrong pane (or nothing) after a close. Now IDs are flat integers (`-p 5`) assigned once and stable for the life of the session.

**Creation commands return IDs** — `attyx tab create` and `attyx split v/h` now print the new pane's ID, so scripts can immediately start targeting it without guessing. Combined with stable IDs, this makes multi-pane automation reliable by default.

**Safer tab close** — `attyx tab close foo` now errors if `foo` doesn't match any tab, instead of silently closing the active tab.
