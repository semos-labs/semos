# Attyx v0.2.32

## Clean `--wait` Output

The `--wait` flag now returns exactly what the command printed — nothing more. Previously, shell integration sequences (like working directory and PATH reports) could leak into the captured stdout, producing unexpected garbage at the start of the output.

This affected all shells (zsh, bash, fish, nushell) when using `--wait` to capture command output. Those internal sequences now go through stderr, keeping your captured stdout clean.
