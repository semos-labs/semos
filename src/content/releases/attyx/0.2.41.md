# Attyx v0.2.41

## Named Keys for `send-keys`

No more memorizing hex codes. You can now send special keys using readable `{NamedKey}` syntax:

```sh
attyx send-keys "ls -la{Enter}"          # type and press Enter
attyx send-keys "{Up}{Enter}"             # rerun last command
attyx send-keys "{Ctrl-c}"                # interrupt
attyx send-keys "{Down}{Down}{Enter}"     # navigate a menu
attyx send-keys "{Escape}:wq{Enter}"      # save and quit vim
```

Full modifier combos work too — `{Ctrl-Shift-Up}`, `{Shift-Tab}`, `{Alt-a}`, `{Shift-F5}`, and more. All keys are case-insensitive. The old C-style escapes (`\n`, `\x03`, etc.) still work if you prefer them.

## Automatic Inter-Key Delay

When you chain named keys like `{Down}{Down}{Enter}`, Attyx now inserts a 30ms delay between each one. This prevents race conditions where TUIs can't redraw fast enough to process rapid-fire input — menu navigation, tab completion, and similar workflows just work.

## Wait for Stable Output

The new `--wait-stable` flag on `send-keys` polls the screen after sending input and waits until the content stops changing, then prints the final output to stdout:

```sh
attyx send-keys --wait-stable "ls{Enter}"            # default 300ms stability window
attyx send-keys --wait-stable 500 "make{Enter}"      # custom 500ms window
```

This replaces the common pattern of `send-keys` → `sleep` → `get-text` with a single command. Especially useful for agent workflows where you need to read back the result of a command.

## `send-text` Is Now an Alias

`send-text` and `send-keys` are now the same command. Existing scripts using `send-text` continue to work — no changes needed.

## Dev Build Skill Isolation

If you're developing Attyx skills, debug builds now use a separate `attyx-dev` skill namespace. This means dev and release skill installations no longer collide.
